## 요구사항
### 1. 시스템 탐험과 분석
- 10개의 API 엔드포인트를 모두 직접 호출해보고 동작을 파악하세요
- 각 API의 목적, 파라미터, 응답을 기록하세요
- 기능 명세서를 참고해서 코드를 읽으며 숨겨진 비즈니스 규칙을 찾아내세요
### 2. 리스크 매트릭스 작성
- 각 기능의 리스크를 비즈니스 임팩트, 변경 빈도, 버그 가능성 세 가지 기준으로 평가하세요.
- 각 기준을 점수화하고, 단순 합산하여 리스크 총점을 계산하세요.
  - 예시) 각 항목을 높음(3점), 보통(2점), 낮음(1점) 으로 잡고 리스크 점수 도출 가능
  - 리스크 점수 = 임팩트 + 빈도 + 가능성
- 기능을 평가하고 우선순위를 매기세요.
### 3. 인수 조건 도출
- 우선순위를 높게 평가한 기능에 대해 5W1H 분석을 수행하세요
- 각 기능의 핵심 인수 조건을 한 문장으로 정의하세요
- 검증해야 할 핵심 포인트들을 나열하세요
### 4. Gherkin 시나리오 작성
- 인수 조건을 Gherkin 형식으로 표현하세요
- 정상 케이스와 예외 케이스를 모두 포함하세요
- 비즈니스 관점에서 이해 가능하게 작성하세요
